## Design Context

### Users
Active runners who use Strava. They want to make running more exciting by putting real money on the line with friends or strangers. They're competitive but social — this isn't hardcore crypto, it's "bet you can't run 50km this month." Context: mobile-first, checking progress mid-day or after a run.

### Brand Personality
**Energetic, Social, Fun** — like challenging your mates to a bet at the pub, but for running. Confident enough to handle real money, playful enough that it never feels like a trading terminal.

### Aesthetic Direction
- **Tone**: Athletic editorial meets modern fintech — bold numbers, clean surfaces, dynamic energy
- **References**: Strava (athletic, data), Monzo/Revolut (trustworthy, micro-interactions), DraftKings (competitive tension, stakes)
- **Anti-references**: Dark crypto dashboards, generic SaaS, glassmorphism, gradient text, neon-on-black
- **Theme**: Light + dark with user toggle. Light mode is primary — differentiates from dark-mode fitness/crypto apps
- **Colors**: Warm coral primary (#FF6B35), emerald for money (#10B981), navy dark mode, warm cream light mode
- **Typography**: Space Grotesk (display/headings — sharp, modern) + Plus Jakarta Sans (body — friendly, readable)
- **Memorable element**: Challenge cards with bold stake callouts and dynamic progress — not generic dashboard cards

### Design Principles
1. **Stakes front and center** — the money and distance are always the loudest elements on screen
2. **Motion = progress** — use animation to reinforce movement and achievement (progress bars, number counters, transitions)
3. **Trust through clarity** — clean money handling UI inspired by Monzo. No ambiguity about what you're staking or winning
4. **Social proof everywhere** — show runners, participants, activity. The app should feel alive, not empty
5. **Athletic, not techy** — typography and color should feel like a sports brand, not a developer tool
